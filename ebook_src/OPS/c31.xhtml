STYLING

With modern applications, developers have become a lot more concerned with having good-looking apps. When Windows Forms was the technology for creating desktop applications, the user interface didn't offer many options for styling the applications. Controls had a standard look that varied slightly based on the operating system version on which the application was running, but it was not easy to define a complete custom look.
随着现代应用程序的发展，开发者越来越关注应用程序的外观。在 Windows Forms 是创建桌面应用程序的技术时，用户界面并没有提供很多用于样式设计的选项。控件具有标准的外观，根据应用程序运行的操作系统版本略有不同，但定义完整的自定义外观并不容易。

This changed with Windows Presentation Foundation (WPF). WPF is based on DirectX and thus offers vector graphics that allow easy resizing of windows and controls. Controls are completely customizable and can have different looks. Styling of applications has become extremely important. An application can have any look. With a good design, the user can work with the application without the need to know how to use a Windows application. Instead, the user just needs to have domain knowledge. For example, the airport in Zurich created a WPF application where buttons look like airplanes. With the button, the user can get information about the position of the plane (the complete application looks like the airport). Colors of the buttons can have different meanings based on the configuration; they can show either the airline or on-time/delay information of the plane. This way, the user of the app easily sees which planes that are currently at the airport have small or big delays.
这种情况随着 Windows Presentation Foundation（WPF）发生了改变。WPF 基于 DirectX，因此提供了矢量图形，允许轻松调整窗口和控件的大小。控件完全可以定制，可以有不同的外观。应用程序的样式变得极其重要。一个应用程序可以拥有任何外观。凭借良好的设计，用户可以在不了解如何使用 Windows 应用程序的情况下使用该应用程序。相反，用户只需要具备领域知识。例如，苏黎世的机场创建了一个 WPF 应用程序，其中按钮看起来像飞机。通过按钮，用户可以获取飞机的位置信息（整个应用程序看起来像机场）。按钮的颜色可以根据配置具有不同的含义；它们可以显示航空公司或飞机的正点/延误信息。这样，应用程序的用户可以轻松看到目前机场中哪些飞机有小延误或大延误。

Having different looks for the app is even more important with modern Windows apps. With these apps, the device can be used by users who haven't used Windows applications before. With users who are knowledgeable of Windows applications, you should think about helping these users be more productive by having the typical process for how the user works easily accessible.
为现代 Windows 应用设计不同的外观变得更加重要。这些应用可以让之前未使用过 Windows 应用程序的用户使用。对于熟悉 Windows 应用程序的用户，应考虑通过让用户的工作流程典型过程易于访问来帮助这些用户提高生产力。

With its guidance on UI design with the Fluent Design System (https://www.microsoft.com/design/fluent/), Microsoft continuously evolves cross-platform UI design for applications, including web, Windows, iOS, Android, macOS, and other cross-platform applications. Many Microsoft applications use guidance from this design, and WinUI plays an important role (https://microsoft.github.io/microsoft-ui-xaml/).
凭借其在 Fluent 设计系统（ https://www.microsoft.com/design/fluent/ ）中关于 UI 设计的指导，微软不断演进跨平台应用程序的 UI 设计，包括网页、Windows、iOS、Android、macOS 以及其他跨平台应用。许多微软应用使用该设计指南，WinUI 发挥着重要作用（ https://microsoft.github.io/microsoft-ui-xaml/ ）。

This chapter starts with the core elements of XAML—shapes that enable you to draw lines, ellipses, and path elements. After that you're introduced to the foundation of shapes—geometry elements. You can use geometry elements to create fast vector-based drawings.
本章从 XAML 的核心元素开始——形状，这些形状使你能够绘制线条、椭圆和路径元素。随后你将了解形状的基础——几何元素。你可以使用几何元素来创建快速基于矢量的绘图。

With transformations, you can scale and rotate any XAML element. With brushes, you can create solid color, gradient, or more advanced backgrounds. You see how to use brushes within styles and place styles within XAML resources.
通过变换，你可以缩放和旋转任何 XAML 元素。通过画刷，你可以创建纯色、渐变或更高级的背景。你将学习如何在样式中使用画刷，并将样式放置在 XAML 资源中。

Finally, with templates you can completely customize the look of controls, and you also learn how to create animations in this chapter.

Shapes are the core elements of XAML. With shapes, you can draw two-dimensional graphics using rectangles, lines, ellipses, paths, polygons, and polylines that are represented by classes derived from the abstract base class Shape. With WinUI, shapes are defined in the namespace Microsoft.UI.Xaml.Shapes.
形状是 XAML 的核心元素。通过形状，你可以使用矩形、线条、椭圆、路径、多边形和多段线绘制二维图形，这些形状由从抽象基类 Shape 派生的类表示。在 WinUI 中，形状定义在命名空间 Microsoft.UI.Xaml.Shapes 中。

Using the code-behind, a geometrical shape can be created using figures and segments. First, you create a two-dimensional array of six points to define three points for the happy state and three points for the sad state (code file Shapes/MainWindow.xaml.cs):
使用代码后台，可以使用图形和段创建几何形状。首先，您创建一个包含六个点的二维数组，以定义快乐状态下的三个点和悲伤状态下的三个点（代码文件 Shapes/MainWindow.xaml.cs ）： 

private readonly Point[,] _mouthPoints = new Point[2, 3]
{
  { new(40, 74), new(57, 95), new(80, 74) },
  { new(40, 82), new(57, 65), new(80, 82) }

Next, you assign a new PathGeometry object to the Data property of the Path. The PathGeometry contains a PathFigure with the start point defined (setting the StartPoint property is the same as the letter M with path markup syntax discussed later in the section “Geometries Using Path Markup”). The PathFigure contains a QuadraticBezierSegment with two Point objects assigned to the properties Point1 and Point2 (the same as the letter Q with two points):
接下来，您将一个新的 PathGeometry 对象分配给 Path 的 Data 属性。 PathGeometry 包含一个带有起点定义的 PathFigure （设置 StartPoint 属性与后面“使用路径标记几何形状”部分中讨论的路径标记语法中的字母 M 相同）。 PathFigure 包含一个带有两个分配给属性 Point1 和 Point2 的 QuadraticBezierSegment 对象（与带有两个点的字母 Q 相同）： 

private bool _laugh = false;
public void SetMouth()
{
  int index = _laugh ? 0: 1;
 
  PathFigure figure = new() { StartPoint = _mouthPoints[index, 0] };
  figure.Segments = new PathSegmentCollection();
  QuadraticBezierSegment segment1 = new()
  {
    Point1 = _mouthPoints[index, 1];
    Point2 = _mouthPoints[index, 2];
  }
 
  figure.Segments.Add(segment1);
  PathGeometry geometry = new();
  geometry.Figures = new PathFigureCollection();
  geometry.Figures.Add(figure);
 
  mouth.Data = geometry;
  _laugh = !_laugh;
}

GEOMETRY    几何形状

The previous sample showed that one of the shapes, Path, uses Geometry for its drawing. You can also use Geometry elements in other places, such as with a DrawingBrush.
之前的示例表明，其中一个形状， Path ，使用 Geometry 进行绘制。你也可以在其他地方使用 Geometry 元素，例如与 DrawingBrush 一起使用。

In some ways, geometry elements are similar to shapes. Just as there are Line, Ellipse, and Rectangle shapes, there are also geometry elements for these drawings: LineGeometry, EllipseGeometry, and RectangleGeometry. There are also big differences between shapes and geometries. A Shape is a FrameworkElement that you can use with any class that supports UIElement as its children. FrameworkElement derives from UIElement. Shapes participate with the layout system and render themselves. The Geometry class can't render itself and has fewer features and less overhead than Shape. The Geometry class directly derives from DependencyObject.
在某些方面，几何元素与形状相似。正如有 Line 、 Ellipse 和 Rectangle 形状一样，这些绘图也有对应的几何元素： LineGeometry 、 EllipseGeometry 和 RectangleGeometry 。形状与几何之间也有显著差异。 Shape 是一个 FrameworkElement ，你可以将其用于任何支持 UIElement 作为其子类的类。 FrameworkElement 派生自 UIElement 。形状参与布局系统并自行渲染。 Geometry 类不能自行渲染，并且比 Shape 具有更少的特性和更低的开销。 Geometry 类直接派生自 DependencyObject 。

The Path class uses Geometry for its drawing. The geometry can be set with the Data property of the Path. Simple geometry elements that can be set are EllipseGeometry for drawing an ellipse, LineGeometry for drawing a line, and RectangleGeometry for drawing a rectangle.
Path 类使用 Geometry 进行绘制。几何形状可以通过 Path 的 Data 属性设置。可以设置的简单几何元素包括：用于绘制椭圆的 EllipseGeometry ，用于绘制线的 LineGeometry ，以及用于绘制矩形的 RectangleGeometry 。

Transforming Using a Matrix
使用矩阵进行变换

Another option for defining multiple transformations at once is to specify a matrix. Here, you use MatrixTransform. MatrixTransform defines a Matrix property that has six values. Setting the values 1, 0, 0, 1, 0, 0 doesn't change the element. With the values 0.5, 1.4, 0.4, 0.5, –200, and 0, the element is resized, skewed, and translated (code file Transformations/MainWindow.xaml):
定义多个变换的另一种方法是指定矩阵。这里你使用 MatrixTransform 。 MatrixTransform 定义了一个 Matrix 属性，它有六个值。设置值为 1, 0, 0, 1, 0, 0 不会改变元素。使用 0.5, 1.4, 0.4, 0.5, –200 和 0 的值，元素会被调整大小、倾斜和移动（代码文件 Transformations/MainWindow.xaml ）： 

<Rectangle Width="120" Height="60" Fill="Gold" Margin="20">
  <Rectangle.RenderTransform>
    <MatrixTransform Matrix="0.5, 1.4, 0.4, 0.5, -200, 0" />
  </Rectangle.RenderTransform>
</Rectangle>

The class MatrixTransform defines the public fields M11, M12, M21, M22, OffsetX, and OffsetY that are set in that order if a string is assigned to the Matrix property. MatrixTransform implements an affine transformation, so only six of the nine matrix members need to be specified. The remaining matrix members have fixed values 0, 0, and 1. The M11 and M22 fields have a default value 1 and are used to scale in the X and Y directions. M12 and M21 have a default value 0 and are used to skew the control. OffsetX and OffsetY have a default value 0 and are used to move the control.
类 MatrixTransform 定义了公共字段 M11 、 M12 、 M21 、 M22 、 OffsetX 和 OffsetY ，如果将字符串分配给 Matrix 属性，这些字段将按此顺序设置。 MatrixTransform 实现仿射变换，因此只需要指定九个矩阵成员中的六个。其余的矩阵成员具有固定的值 0、0 和 1。 M11 和 M22 字段具有默认值 1，用于在 X 和 Y 方向上缩放。 M12 和 M21 具有默认值 0，用于倾斜控件。 OffsetX 和 OffsetY 具有默认值 0，用于移动控件。

Resource Hierarchies    资源层次结构

As you have seen with the styles sample, usually styles are stored within resources. The FrameworkElement base class defines the Resourcces property, so every class that derives from the FrameworkElement base class can specify resources.
正如你在样式示例中所看到的，通常样式存储在资源中。 FrameworkElement 基类定义了 Resourcces 属性，因此每个从 FrameworkElement 基类派生的类都可以指定资源。

Resources are searched hierarchically. If you define the resource with the root element, it applies to every child element. If the root element contains a Grid, the Grid contains a StackPanel, and you define the resource with the StackPanel, then the resource applies to every control within the StackPanel. If the StackPanel contains a Button and you define the resource just with the Button, then this style is valid only for the Button.
资源按层次结构搜索。如果你使用根元素定义资源，它将适用于所有子元素。如果根元素包含 Grid ，而 Grid 包含 StackPanel ，你使用 StackPanel 定义资源，那么该资源将适用于 StackPanel 内部的所有控件。如果 StackPanel 包含 Button ，你仅使用 Button 定义资源，那么此样式仅适用于 Button 。

ANIMATIONS    动画

When you use animations, you can make a smooth transition between images by using moving elements, color changes, transforms, and so on. XAML makes it easy to create animations. You can animate the value of most dependency properties. Different animation classes exist to animate the values of different properties, depending on their type.
使用动画时，可以通过移动元素、颜色变化、转换等方式在图像之间实现平滑过渡。XAML 可以轻松创建动画。你可以对大多数依赖属性的值进行动画处理。根据属性的类型，存在不同的动画类来对不同的属性值进行动画处理。

The most important element of an animation is the timeline. This element defines how a value changes over time. Different kinds of timelines are available for changing different types of values. The base class for all timelines is Timeline. To animate a property of type double, you can use the class DoubleAnimation. The Int32Animation is the animation class for int values. You use PointAnimation to animate points and ColorAnimation to animate colors.

Easing Functions    缓动函数

With the animations you've seen so far, the value changes in a linear way. In real life, a move never happens in a linear way. The move could start slowly and progressively get faster until reaching the highest speed, and then it slows down before reaching the end. When you let a ball fall against the ground, the ball bounces a few times before staying on the ground. Such nonlinear behavior can be created by using easing functions.
在你看到的动画中，值的变化是线性的。在现实生活中，移动永远不会是线性的。移动可能开始缓慢，然后逐渐加速直到达到最高速度，然后在接近结束时减速。当你让球掉到地上时，球会在落地前弹跳几次。这种非线性行为可以通过使用缓动函数来创建。

Animation classes have an EasingFunction property. This property accepts an object that derives from the base class EasingFunctionBase. With this type, an easing function object can define how the value should be animated over time. Several easing functions are available to create a nonlinear animation. Examples include ExponentialEase, which uses an exponential formula for animations; QuadraticEase, CubicEase, QuarticEase, and QuinticEase, with powers of 2, 3, 4, or 5; and PowerEase, with a power level that is configurable. Of special interest are SineEase, which uses a sinusoid curve; BounceEase, which creates a bouncing effect; and ElasticEase, which resembles animation values of a spring oscillating back and forth.
动画类有一个 EasingFunction 属性。此属性接受一个从基础类 EasingFunctionBase 派生的对象。使用这种类型，缓动函数对象可以定义值应如何随时间变化。有多种缓动函数可用于创建非线性动画。例如 ExponentialEase 使用指数公式进行动画； QuadraticEase 、 CubicEase 、 QuarticEase 和 QuinticEase 使用 2、3、4 或 5 次方； PowerEase 的幂级数可配置。特别值得关注的是 SineEase 使用正弦曲线； BounceEase 创建弹跳效果； ElasticEase 类似于弹簧来回振荡的动画值。

To make it easier for you to create animated user interfaces, UWP apps define transitions. Transitions make it easier to create compelling apps without the need to think about what makes a cool animation. Transitions predefine animations for adding, removing, and rearranging items in a list; opening panels; changing the content of content controls; and more.
为了帮助您更轻松地创建动画用户界面，UWP 应用定义了过渡效果。过渡效果使您无需考虑什么构成酷炫动画就能创建引人入胜的应用。过渡效果预定义了用于添加、删除和重新排列列表中的项目、打开面板、更改内容控件的内容等动画。

VISUAL STATE MANAGER    视觉状态管理器

Earlier in this chapter in the section “Control Templates,” you saw how to create control templates to customize the look of controls. Something was missing there. With the default template of a button, the button reacts to mouse moves and clicks and looks differently when the mouse moves over the button or the button is clicked. This change in the look of a control is handled with the help of visual states and animations, controlled by the VisualStateManager.
在本章“控件模板”部分，你了解了如何创建控件模板来自定义控件的外观。但那里缺少了一些东西。使用按钮的默认模板时，按钮会对鼠标移动和点击做出反应，并在鼠标移动到按钮上或按钮被点击时显示不同的外观。控件外观的变化是通过视觉状态和动画处理的，这些由 VisualStateManager 控制。

This section examines changing the button style to react to mouse moves and clicks, but it also describes how to create custom states to deal with changes of a complete page when several controls should switch to the disabled state—for example, when some background processing occurs.
本节将探讨如何使按钮样式对鼠标移动和点击做出反应，同时也描述了如何创建自定义状态来处理多个控件应切换到禁用状态时整个页面的变化——例如，在后台处理发生时。

With a XAML control, visual states, state groups, and states can be defined that specify animations for a state. State groups exist to allow having multiple states at once. For one group, only one state is allowed at one time. However, another state of another group can be active at the same time. Examples for this are the states and state groups with a button. The Button control defines the state groups CommonStates and FocusStates. States defined with FocusStates are Focused, Unfocused, and PointerFocused. The CommonStates group defines the states Normal, Pressed, Disabled, and PointerOver. With these options, multiple states can be active at the same time, but there is always only one state active within a state group. For example, a button can be in focus and in the normal state. It can also be in focus and pressed. You can also define custom states and state groups.
使用 XAML 控件，可以定义视觉状态、状态组和状态，这些状态会指定相应状态的动画。状态组的存在是为了允许同时存在多个状态。对于一组状态，同一时间只允许一个状态处于活动状态。然而，另一组中的另一个状态可以同时处于活动状态。按钮的状态和状态组就是这种情况的例子。控件 Button 定义了状态组 CommonStates 和 FocusStates 。使用 FocusStates 定义的状态有 Focused 、 Unfocused 和 PointerFocused 。状态组 CommonStates 定义了状态 Normal 、 Pressed 、 Disabled 和 PointerOver 。通过这些选项，可以同时激活多个状态，但在状态组内始终只有一个状态处于活动状态。例如，一个按钮可以处于焦点状态和正常状态。它也可以处于焦点状态和按下状态。你也可以定义自定义状态和状态组。

Let's get into concrete examples.
让我们来看一些具体的例子。
